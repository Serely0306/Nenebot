# Nenebot Workspace

`Nenebot` 是当前整套 Bot 系统的本地维护工作区与部署工作目录。  
用于集中维护多个相关组件、脚本、配置与联动逻辑

当前系统仍以**整仓维护 / 整仓部署**为主：  
服务器侧通常通过对整个工作区执行 `git pull` 来同步更新。

---

## 1. 工作区定位

这个工作区主要承担以下作用：

- 多个 Bot 组件的统一维护目录
- 当前线上部署所依赖的完整工作目录
- 跨项目联调与本地开发场地
- 配套工具、上传服务、抓包工具、外部 API 服务的整合入口

从维护角度看，当前工作区中的内容大致可以分为：

1. **正式运行组件**
2. **配套服务与辅助工具**

---

## 2. 当前系统分层

### 2.1 接入与转发层

- `OneBotFilter`
  - 当前消息接入与核心转发层
  - 承接 QQ / OneBot 消息流，并向下游 Bot 分发
  - 在转发之外也承载了一部分核心功能

### 2.2 Bot 功能层

当前工作区中涉及的 Bot 侧主要包括：

- `harukineo`：分布式部署，核心sekai功能承载
- `lunabot`：基于开源代码进行配置和本地部署，并进行功能扩展
- `zerobot`（相关代码当前主要位于 `ZeroBot-Plugin-master`），基于开源代码进行配置部署，修改原功能和新增插件
- `AstrBot`（当前未启用）


### 2.3 数据与辅助服务层

- `Nenebot_tools`
  - 为 `lunabot` 提供数据上传相关后端
  - 承载拉群申请相关 API
  - 联通 `OneBotFilter` 与 QQ 应用侧，用于通知与自动审批

- `catcher`
  - 面向 Android 用户的抓包工具
  - 用于抓取游戏数据并上传到 `Nenebot_tools`
  - 上传后的数据再供 `lunabot` 调用

- `haruki-tool`
  - 开源项目
  - 本地配置部署后，为 `lunabot` 提供游戏内通信相关 API
